# Direction

## Cut scheduling latency #direction #now #performance

The current priority is reducing booking-confirmation latency. Work that makes
the booking path faster or more reliable moves this forward.

## Open the public API #direction #api

We are stabilizing a public scheduling API for partners. Favor additive,
versioned changes and clear contracts.

## Reduce on-call pain #direction #reliability

Cut noisy alerts and flaky tests. Reliability work counts as feature work this
quarter.
